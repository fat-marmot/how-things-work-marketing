The honest version.

This app is a personal hobby project. It tracks nothing that doesn't earn its keep. What follows is the same thing said two ways — first plainly, then as the disclaimer that has to exist for anyone using it for anything serious.

Last updated · 26 April 2026

What's collected

The app is designed to know as little about you as possible. There's no account, no sign-in, no analytics SDK, no tracking pixel, no advertising. I haven't installed any of that and I'm not planning to. If that ever changes, this page changes too.

The API behind the app sees two things to keep the service running:

  • Your IP address, the same way every web server sees it.
  • An anonymous device ID generated by the app on first launch — a random UUID, not tied to your name, email, Apple ID, or anything else identifying.

Both are used for rate limiting, so the service isn't slammed by abuse and I don't end up with a surprise cloud bill. They live in a short-term cache and aren't stored long-term — with one exception: when you submit thumbs-up or thumbs-down feedback on a topic, the device ID (not your IP) is saved with that feedback row, so a single device can't spam votes.

The web server keeps short-lived access logs like every web server does. They contain timestamps, paths, and IPs. They roll over.

What I can see at the aggregate level

I haven't built any analytics dashboards, but a couple of dashboards exist that I didn't have to set up:

  • Apple's App Store dashboard shows me download counts, session counts, and crash reports. None of it is per-user trackable. Hopefully this number is at least three (my wife and kids).
  • The DigitalOcean Spaces dashboard shows me how often images get fetched. Aggregate, no per-user data.

That's the entire visibility I have into who's using the app. I've never deployed an iOS app before this, so there may be App Store metrics I haven't found yet — if I find anything else worth mentioning, I'll add it here.

Anonymous use is the model

No account is needed to read the content. Download the app, open it, read. That's the entire onboarding flow and that's how it's intended to stay.
